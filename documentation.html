Proxmox-LDC Documentation

Introduction

Proxmox-LDC is a desktop client for managing Proxmox servers. It provides an intuitive graphical interface for managing virtual machines, containers, and server resources.

Getting Started

Initial Setup

  1. Launch the application
  2. Follow the setup wizard to configure your Proxmox server connection
  3. Enter your Proxmox server URL, username, and password
  4. Trust the server certificate when prompted

Adding Multiple Servers

You can add multiple Proxmox servers during setup or later in Server Settings. Use the server dropdown on the home dashboard to switch between servers.

Features

Dashboard

The home dashboard provides an overview of your Proxmox server, including:

  • Server statistics (CPU, memory, disk usage)
  • List of virtual machines with status indicators
  • List of containers with status indicators
  • Quick actions for VMs and containers

Virtual Machine Management

Access via Virtual Machines → Manage virtual machines:

  • View all VMs with their current status
  • Start, stop, and restart VMs
  • Open VM console (SPICE/VNC) in separate windows
  • View detailed VM information and statistics
  • Search and sort VMs

Container Management

Access via Containers → Manage Containers:

  • View all containers with their current status
  • Start, stop, and restart containers
  • Open container console via SSH
  • View detailed container information and statistics
  • Search and sort containers

Creating Virtual Machines

Access via Virtual Machines → Add a virtual machine:

Use the wizard to create new VMs with options for:

  • VM name and ID
  • Node selection
  • CPU and memory allocation
  • Storage configuration
  • Network settings
  • ISO image selection

Server Settings

Access via Settings → Server Settings:

  • View and edit server connection details
  • Add new servers
  • Remove servers
  • Manage certificate trust
  • Switch between multiple servers

App Settings

Access via Settings → App Settings:

  • Configure custom configuration folder location
  • Manage application preferences

Shell Access

Access via Settings → Shell:

Provides SSH terminal access to your Proxmox host for advanced operations.

Console Access

VM Consoles

Virtual machines can be accessed via SPICE or VNC consoles, which open in separate windows using remote-viewer (virt-viewer).

Note: VMs configured with external displays cannot be accessed via console.

Container Consoles

Containers are accessed via SSH terminal. The app will:

  1. Open a terminal window
  2. Connect via SSH to your Proxmox host
  3. Automatically enter the container using pct enter

You will be prompted for your SSH password (same as your Proxmox API password).

Tips and Troubleshooting

Console Not Working

  • VMs: Ensure the VM is running and has SPICE/VNC enabled
  • Containers: Ensure SSH is enabled on your Proxmox host and the container is running
  • Install virt-viewer for VM console access: sudo apt install virt-viewer

Certificate Errors

If you see certificate errors, go to Settings → Server Settings and click "Trust Server Certificate".

Data Not Updating

Use the "Refresh" button in the Manage VMs/Containers view, or refresh the dashboard to get the latest data from Proxmox.

Keyboard Shortcuts

  • Use the menu bar to navigate between different views
  • Search boxes allow filtering of VMs and containers
  • Sort options are available in the Manage views
